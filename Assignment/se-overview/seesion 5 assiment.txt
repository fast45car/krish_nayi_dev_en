1. Write the Git command for each: initialize a repo, stage all files, commit with a message, push to remote, pull latest changes.


Initialize a repo:      git init
Stage all files:        git add .
Commit with a message:  git commit -m "your message"
Push to remote:         git push origin main
Pull latest changes:    git pull origin main

| Task | Command |
|---|---|
| Initialize a repo | `git init` |
| Stage all files | `git add .` |
| Commit with a message | `git commit -m "your message"` |
| Push to remote | `git push origin main` |
| Pull latest changes | `git pull origin main` |


2. What is the difference between git fetch and git pull? Answer in 2 lines.What is the difference between git fetch and git pull? Answer in 2 lines.

git fetch downloads changes from the remote repository without merging them into your local branch, 
while git pull fetches changes and automatically merges them into your current branch. 
In essence, git pull is a combination of git fetch followed by git merge.

3.Create a new GitHub repository, push any small text file from your local system, and share the repo link

Step 1: Create a new repository on GitHub (via browser)

Go to github.com/new

Name it (e.g., test-repo)

Leave it Public or Private (your choice)

Do NOT initialize with README (keep it empty)

Click Create repository

Step 2: Create a small text file local
git init
git add sample.txt
git commit -m "Add sample text file"
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
git push -u origin main
Step 4: Get your repo link

Your repository link will be: https://github.com/YOUR_USERNAME/YOUR_REPO_NAME



4.What is the difference between a Fork and a Pull Request? Answer in 2 lines.

Fork – A server-side copy of someone else's repository under your own GitHub account, which exists independently of the original.
Pull Request – A notification-based proposal that asks the maintainers of the original repository to 
review and merge specific changes from your fork (or branch) back into their project.
